### 一、题目

> Suppose there are n facilities and m customers. We wish to choose:
> 1. which of the n facilities to open
> 2. the assignment of customers to facilities

> The objective is to minimize the sum of the opening cost and the assignment cost.
> **Note:** The total demand assigned to a facility must not exceed its capacity.

### 二、分析&解题

对问题进行数学建模：  
设 ***I*** 为备选的设施集合  
***J*** 为服务对象集合  
对于每个设施**f<sub>i</sub> ∈ *I*** ，其具有两个属性：
- **Capacity：**设施 ***i*** 的最大生产能力
- **OpenCost：**建设设施 ***i*** 所需要的费用

对于每个订单**c<sub>j</sub> ∈ *J***，其具有两个属性：
- **Demand：**该顾客的需求量
- **AssignmentCost：**可理解为设施 ***i*** 送货给顾客 ***j*** 的运费

于是问题就变为对于给定的候选建设设施集合和需完成的订单集合，  
求如何在设施的能力范围之内，将订单分配给各设施去完成，  
使得**总费用**最小
这里的费用包括两部分：
- 所有设施的建设成本
- 所有订单的运费

#### 解法一：贪心算法


上面说到费用包括两部分，那么自然而然地贪心也可以分为两个方向，即
- 对建设成本进行"贪心"
- 对运费进行"贪心"

考虑到一般情况，订单数量往往远大于设施数量  
故而对后者进行"贪心"带来的效益比较大

具体而言，即：
1. 对每一个用户，将所有设施按对其服务的运费排序
2. 优先选择运费少的设施判断其剩余容量能否为该用户服务，若能则指派其为该用户服务，否则判断下一设施

Python实现：
``` python
import time

# 设施数量和顾客数量
facilitiesNum = 0
customersNum = 0
# capacity[i]/openCost[i]: 第i个设施的容量/建设成本
capacity = []
openCost = []
# demand[j]: 第j个顾客的需求量
demand = []
# assignmentCost[j][i]: 第i个设施为第j个用户服务所需费用
assignmentCost = []

# 运费对象：某一设施服务特定顾客所需未用
# 用于贪心算法求解时的指派费用排序
class Freight:
    def __init__(self, facilityID, cost):
        self.facilityID = facilityID
        self.cost = cost
    def __lt__(self, other):
        return self.cost < other.cost

def greedy():
    # 最小总费用
    result = 0
    # assignment[i]: 是否启用设施i，1-启用, 0-关闭
    facilities = [0] * facilitiesNum
    # assignment[j]: 为用户j服务的设施序号
    assignment = [-1] * customersNum
    _assignmentCost = assignmentCost.copy()
    _capacity = capacity.copy()
    for j in range(customersNum):
        # 按"运费"从小到大将设施排序
        _assignmentCost[j].sort()
        # 这里k并不一定等于设施id
        for k in range(facilitiesNum):
            freight = _assignmentCost[j][k]
            # 若该设施剩余容量还能满足该客户需求，则指派该用户到这个设施
            if _capacity[freight.facilityID] >= demand[j]:
                # 若未启用，则启用
                if facilities[freight.facilityID] == 0:
                    facilities[freight.facilityID] = 1
                    result += openCost[freight.facilityID]
                result += freight.cost
                _capacity[freight.facilityID] -= demand[j]
                assignment[j] = freight.facilityID
                break
    return result, facilities, assignment

# 处理输入数据
def loadData(raw):
    raw = raw.split('\n')
    temp = raw[0].split()
    # 获取设施数量和顾客数量
    global facilitiesNum
    global customersNum
    facilitiesNum = int(temp[0])
    customersNum = int(temp[1])
    # 设施容量和建设成本
    for i in range(1, facilitiesNum + 1):
        temp = raw[i].split()
        capacity.append(int(temp[0]))
        openCost.append(int(temp[1]))
    # 顾客需求量和所有运费
    demandAndAssignmentCost = " ".join(raw[facilitiesNum + 1:]).split()
    for j in range(customersNum):
        demand.append(int(demandAndAssignmentCost.pop(0).rstrip('.')))
    for j in range(customersNum):
        temp = []
        for i in range(facilitiesNum):
            temp.append(Freight(i, int(demandAndAssignmentCost.pop(0).rstrip('.'))))
        assignmentCost.append(temp)

# 清空数据
def cleanData():
    capacity.clear()
    openCost.clear()
    demand.clear()
    assignmentCost.clear()

if __name__ == '__main__':
    with open('greedyOutput.txt', 'w') as outfile:
        for i in range(1,72):
            filename = 'p' + str(i)
            with open("Instances/" + filename) as infile:
                data = infile.read()
            loadData(data)
            startTime = time.time()
            result, facilities, assignment= greedy()
            endTime = time.time()
            totalTime = endTime - startTime
            print(filename + ':', file = outfile)
            print("Time: %.6fs" % (totalTime), file = outfile)
            print("Minimum cost:", result, file = outfile)
            print("Status of facilities:", facilities, file = outfile)
            print("The assignment of customers to facilities:", assignment, file = outfile)
            print('\n', file = outfile)
            cleanData()

```

#### 解法二：模拟退火算法

关于模拟退火的介绍就不多说，网上有很多  
应用在该问题的主要实现思路如下：
1. 给定初温`T0`、内循环次数`m`、初始解（这里是随机生成）
2. 判定当前温度`Ti`是否达到终止温度（这里选用`1.0`），是转5，否则继续
3. 进入内循环，每次循环：
    1. 从当前解中随机抽取一位顾客`j`以及为其服务的设施`i`
    2. 从其余设施中随机抽取一个能满足该顾客需求的设施`k`
    3. 将为顾客`j`服务的设施由`i`更换为`k`，得到新解
    4. 根据*Metropolis准则*判断是否接受新解，是则替换当前解为新解，否则进入下一次内循环
4. 更新`Ti`，这里是根据快速模拟退火算法的降温方式进行降温，转2
5. 输出近似最优解

其中：
1. *Metropolis准则*：
   ![Metropolis](/img/in-post/Algorithm/Project/Metropolis.png)
   `E(x)`可理解为目标函数，`T`为当前温度，由此生成接受（新解的）概率`p`,其随着`T`下降而下降，模拟物理退火过程
2. *快速模拟退火算法的降温方式*：
   ![fastSA](/img/in-post/Algorithm/Project/fastSA.png)
   其中`T0`代表初温，`t`代表时刻，`T(t)`为时刻`t`的温度

注意：需要将[71个样例](https://pan.baidu.com/s/16HrBlKS7-X5MuDeRXUwAAA)（提取码：rbtw）放在代码所在目录的`Instances/...`下才可运行  
输出存储在同级目录下的`simulatedAnnealingOutput.txt`

Python实现：
``` python
import time
import random
import math

# 设施数量和顾客数量
facilitiesNum = 0
customersNum = 0
# capacity[i]/openCost[i]: 第i个设施的容量/建设成本
capacity = []
openCost = []
# demand[j]: 第j个顾客的需求量
demand = []
# assignmentCost[j][i]: 第i个设施为第j个用户服务所需费用
assignmentCost = []

def getRandomResult():
     # 近似最小总费用
    result = 0
    # assignment[i]: 是否启用设施i，1-启用, 0-关闭
    facilities = [0] * facilitiesNum
    # assignment[j]: 为用户j服务的设施序号
    assignment = [-1] * customersNum
    remainCapacity = capacity.copy()
    for j in range(customersNum):
        # 过滤出能服务该用户的措施
        available = []
        for i in range(facilitiesNum):
            if remainCapacity[i] >= demand[j]:
                available.append(i)
        # 从可用列表中随机抽取一个设施来服务该用户
        assignment[j] = random.choice(available)
        remainCapacity[assignment[j]] -= demand[j]
        result += assignmentCost[j][assignment[j]]
        # 若未启用该设施，则启用
        if facilities[assignment[j]] == 0:
            facilities[assignment[j]] = 1
            result += openCost[assignment[j]]
    return result, facilities, assignment, remainCapacity

# 根据给定解生成下一随机解
def generateNextResult(result, facilities, assignment, remainCapacity):
    # 随机选择一位已经指派任务的顾客及为其服务的设施
    # 将选定用户的指派设施替换成另一个随机选择的能满足该用户需求的设施
    while(True):
        customer = random.choice(range(customersNum))
        oldFacility = assignment[customer]
        # 过滤出其余能服务该用户的措施
        available = []
        for i in range(facilitiesNum):
            if remainCapacity[i] >= demand[customer] and i != oldFacility:
                available.append(i)
        # 若所有设施均不可用，则寻找下一位顾客
        if len(available) == 0:
            continue
        # 从可用设施里随机选取一个
        newFacility = random.choice(available)
        # 重新计算两个设施的剩余容量
        remainCapacity[newFacility] -= demand[customer]
        remainCapacity[oldFacility] += demand[customer]
        assignment[customer] = newFacility
        # 若老设施无其他顾客继续使用，则关闭
        if remainCapacity[oldFacility] == capacity[oldFacility]:
            facilities[oldFacility] = 0
            result -= openCost[oldFacility]
        # 若新设施未启用，则启用
        if facilities[newFacility] == 0:
            facilities[newFacility] = 1
            result += openCost[newFacility]
        result += assignmentCost[customer][newFacility] - assignmentCost[customer][oldFacility]
        return result, facilities, assignment, remainCapacity

# 初温，内循环次数
def simulatedAnnealing(T0, innerNum):
    # 第一次降温
    t = 0
    # 冷却进度表，采用快速降温方式
    Ti = T0 / (1 + t)
    # 随机初始解
    result, facilities, assignment, remainCapacity = getRandomResult()
    # 终止条件：温度小于1.0
    while(Ti >= 1.0):
        for i in range(innerNum):
            # 获取下一随机解
            t_result, t_facilities, t_assignment, t_remainCapacity = generateNextResult(result, facilities.copy(), assignment.copy(), remainCapacity.copy())
            # 新解比旧解差，根据Metropolis准则判断是否接受新解
            if t_result >= result:
                p = math.exp((result - t_result) / Ti)
                # 不接受
                if p < random.random():
                    continue
            # 接受新解
            result, facilities, assignment, remainCapacity = t_result, t_facilities.copy(), t_assignment.copy(), t_remainCapacity.copy()
        # 降温
        t += 1
        Ti = T0 / (1 + t)
    # 近似最优解
    return result, facilities, assignment

# 处理输入数据
def loadData(raw):
    raw = raw.split('\n')
    temp = raw[0].split()
    # 获取设施数量和顾客数量
    global facilitiesNum
    global customersNum
    facilitiesNum = int(temp[0])
    customersNum = int(temp[1])
    # 设施容量和建设成本
    for i in range(1, facilitiesNum + 1):
        temp = raw[i].split()
        capacity.append(int(temp[0]))
        openCost.append(int(temp[1]))
    # 顾客需求量和所有运费
    demandAndAssignmentCost = " ".join(raw[facilitiesNum + 1:]).split()
    for j in range(customersNum):
        demand.append(int(demandAndAssignmentCost.pop(0).rstrip('.')))
    for j in range(customersNum):
        temp = []
        for i in range(facilitiesNum):
            temp.append(int(demandAndAssignmentCost.pop(0).rstrip('.')))
        assignmentCost.append(temp)

# 清空数据
def cleanData():
    capacity.clear()
    openCost.clear()
    demand.clear()
    assignmentCost.clear()

if __name__ == '__main__':
    with open('simulatedAnnealingOutput.txt', 'w') as outfile:
        for i in range(1,72):
            filename = 'p' + str(i)
            with open("Instances/" + filename) as infile:
                data = infile.read()
            loadData(data)
            startTime = time.time()
            result, facilities, assignment = simulatedAnnealing(100, 100)
            endTime = time.time()
            totalTime = endTime - startTime
            print(filename + ':', file = outfile)
            print("Time: %.6fs" % (totalTime), file = outfile)
            print("Minimum cost:", result, file = outfile)
            print("Status of factories:", facilities, file = outfile)
            print("The assignment of customers to facilities:", assignment, file = outfile)
            print('\n', file = outfile)
            cleanData()
```

从算法流程上来看，模拟退火算法的性能很大取决于参数（初温、初始解等）的设置以及三个函数（状态产生函数、状态接受函数、温度更新函数）的设计

这里有比较明显的两点可以获取更优解
1. 将贪心算法的解作为模拟退火算法的初始解
2. 增大内循环次数
